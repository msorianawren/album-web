# Engineering Documentation

This directory is the canonical home for operating and maintaining the project.

- `architecture/` records stable system boundaries and decisions.
- `operations/` contains release, incident, and rollback runbooks.
- `maintenance/` records repository hygiene inventories and cleanup decisions.
- `reports/` is reserved for dated, historical investigation evidence.

Do not add new engineering reports to the repository root. Update a canonical
document when it describes an ongoing system rule; put one-time evidence in a
dated report under `reports/`.
